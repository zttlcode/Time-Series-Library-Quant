这套代码是一个**完整的量化交易信号过滤实验平台**，核心任务是：给你一堆“买入信号”（比如MACD、KDJ金叉），模型要判断哪些信号真的值得做，哪些该跳过，从而提高交易表现。

我用通俗的“流水线”方式，分步骤解释它**从头到尾怎么跑**。

## 使用方式

#### 1、使用 ERDF 标签
```python
label_method = 'erdf_evt'
good_label_threshold = 0.4
run_tag = 'ERDF_MACD_KDJ_A'
```

#### 2、使用固定局部极值标签
```python
label_method = 'fixed_extrema'
good_label_threshold = 0.4
fixed_extrema_win = 3
run_tag = 'FIXED_EXTREMA_MACD_KDJ_A'
```

#### 3、使用未来收益标签
```python
label_method = 'future_return'
future_return_threshold = 0.02
run_tag = 'FUTURE_RETURN_2PCT_MACD_KDJ_A'
```

#### 4、使用 Triple Barrier 训练 BiLSTM 模型
```python
label_method = 'triple_barrier_meta'
tb_horizon = 60
tb_pt_atr = 1.5
tb_sl_atr = 1.0
run_tag = 'TB_META_MACD_KDJ_A'
```

此时你的结果会自动比较：
```text
Meta-Label Model trained by selected label
Original Signal
Triple Barrier + XGB
Buy and Hold
```

### 1. 配置：你想做什么实验？
代码一开头有个 `Config` 类，就像实验的“控制面板”，你可以在里面选择：
- 用什么交易策略产生的信号（例如 `MACD_KDJ`）
- 用什么标注方法给信号打“好坏标签”（例如 `fixed_extrema` 局部极值）
- 模型参数、训练轮数、交易成本、要不要多随机种子跑等。

这个配置文件决定了整个实验跑什么、怎么跑。

---

### 2. 数据准备：把原始行情变成特征
`load_and_prepare` 函数负责把原始 OHLCV 数据和交易信号文件读到一起，然后计算出各种技术指标（均线、MACD、RSI、KDJ、ATR、收益率等），形成一个**特征矩阵**。

这一步的输出是：
- 一个 DataFrame，每一行是一个交易日，列是各种技术指标 + 当日信号（buy/sell/none）
- 一个特征列名单（后续模型要用哪些列）

---

### 3. 标注：给每个买入信号贴上“好坏”标签
这是整个实验最关键的环节。`build_labeler` 会根据配置里的 `label_method`，给历史上每个买入信号生成两个东西：
- `label_value`：一个连续值（例如距离局部极值的距离，越小越好）
- `meta_label`：一个**二分类标签**（1 = 好信号，0 = 坏信号）

比如用 `fixed_extrema` 时，信号离局部低点越近，`label_value` 越小，若小于 `good_label_threshold` 则标记为好信号（`meta_label=1`）。

标注完以后，模型就能学习“什么样的特征窗口对应好信号，什么样的对应坏信号”。

---

### 4. 数据划分：时间序列切分训练/验证/测试
`split_signals_train_val_test` 按时间顺序把信号分成三段：
- 训练集（前 60%）：用来训练模型
- 验证集（中间 20%）：用来早停和**阈值校准**
- 测试集（后 20%）：用来最终评估，不参与任何训练或阈值选择

这里还做了**排毒（purging）**：确保训练集信号的未来窗口不会碰到验证集，防止信息泄露。

---

### 5. 构建数据集和全局标准化
这是一个**多股票联合训练**的框架：
1. 第一遍扫描所有股票，只用训练期的数据拟合一个**全局标准化器**（StandardScaler），这样所有股票的特征尺度就统一了。
2. 第二遍利用这个 scaler，为每只股票创建训练、验证、测试的 `SignalDataset`，并合并成一个跨股票的大训练集。

`SignalDataset` 输出的每个样本是：
- 信号日前 `window_size=160` 天的特征矩阵（形状 160×特征数）
- 该信号的 `meta_label`（好/坏）
- 同一窗口的绝对收益率（备用）

训练时还可以启用**均衡采样**，解决“好信号太少”导致的模型偷懒问题。

---

### 6. 模型训练：序列分类器
模型 `MetaLabelSequenceClassifier` 是一个 BiLSTM 编码器，读取 160 天的特征序列，输出一个 `p_good`（该信号值得做的概率）。

训练过程：
- 用加权二元交叉熵损失（防止类别不平衡）
- 每个 epoch 在验证集上算 AUC/AP 等指标
- 如果验证集指标不再提升，就早停
- 保存最佳模型

---

### 7. 阈值校准：决定 p_good 多高才执行
模型输出的是连续概率，但回测时需要二值决策（执行/跳过）。这一步在**验证集**上完成，找到一个最优阈值 `global_threshold`：
- 遍历候选阈值，对每个阈值计算执行后的精确率、召回率、F1 等
- 限制执行覆盖率在合理范围（比如 20%~45%），选 F1 最高的阈值
- 如果找不到满足条件的，就用分位数兜底

这个阈值会在测试集回测时固定使用。

---

### 8. 外部基线：Triple Barrier + XGBoost
除了深度模型，框架还实现了传统 Meta-Labeling 基线：
- 用**三重障碍法**给训练集和验证集的买入信号重新打标签（未来先碰止盈还是止损）
- 训练一个 XGBoost 分类器（用同样的窗口统计特征）
- 在验证集上用同样方法校准 XGB 的阈值

这样最后对比时，就有了一个非深度学习的强基线。

---

### 9. 测试集回测 & 四个策略比较
对每只股票的测试集，按不同的交易成本（0, 5, 10, 20 bps），运行四种策略：

| 策略 | 含义 |
|------|------|
| **Original Signal** | 原始信号来了就买，不做任何过滤 |
| **Meta-Label Model** | 用训练好的 BiLSTM 模型，只有 `p_good >= 阈值` 才买 |
| **Triple Barrier + XGB** | 用 XGBoost 模型过滤后的信号 |
| **Buy & Hold** | 测试初期买入一直持有到期末，作为市场基准 |

回测逻辑很简单：初始资金 1.0，买入时扣成本，卖出时全平仓，一直算净值曲线。如果信号是 `sell` 且有持仓就平仓，否则继续等下一个 `buy`。

---

### 10. 输出结果：指标、图表、统计检验
最终对每个组合（股票×成本×策略）计算：
- 夏普比率
- 最大回撤（MDD）
- 累计收益率
- 模型执行覆盖率（多少买入信号被实际执行了）

然后汇总成表格，并做两件事：
- **绘图**：画出每只股票和所有股票平均的累计收益曲线
- **显著性检验**：用 Wilcoxon 符号秩检验 + Bootstrap 置信区间，判断模型相比原始信号、相比 XGB 的改善是否统计显著

如果开启了多随机种子，还会把所有种子的结果合并，输出均值和标准差，增强结论的稳健性。

---

### 一句话总结
**这个实验平台就是：用历史数据训练一个深度学习过滤器，让它学会在原始买入信号中“挑好货”，然后和不过滤、传统XGB过滤以及买入持有策略比谁赚得多、回撤小，并用量化指标和统计检验证明方法有效。**

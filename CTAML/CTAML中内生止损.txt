## 一、内生止损在创新点中的地位

论文摘要明确指出 CTAML 包含“基于 \(\hat{\xi}\) 的内生止损止盈”，§3.5 也详述了利用预测的 GPD 形状参数 \(\hat{\xi}\) 动态设定止损宽度，解决了原始方案中参数 \(\kappa\) 同时出现在距离和止损中的循环论证问题。
**这是方法论上的闭环**，属于理论贡献而非单纯的工程技巧。但在当前代码和实验中，你的主要实证目标是：
1. 证明 ERDF 标签的有效性；
2. 验证 CTAML 在时间序列外推下的过滤能力。

你已经在单股票（AES）上获得了过滤带来的夏普提升和回撤降低，这足以支撑论文的核心论点。内生止损止盈的加入，对已经盈利的 AES 信号（买入后价格上涨，止损根本不会触发）不会改变夏普；对于可能亏损的信号，止损会降低回撤但可能也会提高交易频率，对最终夏普的影响方向是不确定的。
因此，**当前不加止损，对你已有的优秀实验结果没有任何负面影响**。

## 二、如果加入止损，代码加在哪里，对夏普的可能影响

若一定要实现，修改只需在 `apply_filter_and_trade` 中：

## 四、若坚持要加，给出具体代码位置

修改 `apply_filter_and_trade`，使其返回除权益曲线外的额外信息，并加入止损逻辑。修改后的函数概要如下（只展示关键改动）：

```python
def apply_filter_and_trade(df, model, cfg, scaler, feature_cols):
    signal_idx = df[df['signal'] != 'none'].index
    capital = 1.0
    position = 0
    equity = [capital]
    signal_records = []
    entry_price = 0.0
    stop_loss_level = None

    for idx, row in df.iterrows():
        if idx in signal_idx:
            i_loc = df.index.get_loc(idx)
            if i_loc >= cfg.window_size:
                x_win = scaler.transform(df[feature_cols].iloc[i_loc-cfg.window_size:i_loc].values)
                x_ten = torch.tensor(x_win, dtype=torch.float32).unsqueeze(0).to(cfg.device)
                abs_ret = torch.tensor(df['abs_returns'].iloc[i_loc-cfg.window_size:i_loc].values,
                                       dtype=torch.float32).unsqueeze(0).to(cfg.device)
                with torch.no_grad():
                    D_hat, xi_out, _ = model(x_ten, abs_ret,
                                            torch.zeros(1).to(cfg.device),
                                            torch.tensor([1.0]).to(cfg.device))
                    d_mid = D_hat[0,1].item()
                    xi_pred = xi_out.item()  # 预测的ξ

                if row['signal'] == 'buy':
                    executed = (d_mid < cfg.D_threshold)
                    signal_records.append({
                        'time': idx,
                        'price': row['close'],
                        'd_mid': d_mid,
                        'executed': executed
                    })
                    if executed:
                        position = capital / row['close']
                        entry_price = row['close']
                        capital = 0
                        # 计算止损位
                        sigma_hat = abs_ret.std().item() + 1e-6
                        stop_width = cfg.stop_loss_c * sigma_hat * np.exp(xi_pred)
                        stop_loss_level = entry_price - stop_width

                elif row['signal'] == 'sell' and position > 0:
                    capital = position * row['close']
                    position = 0
                    stop_loss_level = None

        # 日内止损检查（在非信号日也检查）
        if position > 0 and stop_loss_level is not None:
            if row['close'] <= stop_loss_level:
                # 触发止损
                capital = position * row['close']
                position = 0
                stop_loss_level = None
                # 可选记录止损事件

        if position > 0:
            equity.append(position * row['close'])
        else:
            equity.append(capital)

    return equity, signal_records
```

**注意**：这仅是一个最简实现，未考虑跳空、滑点、多空双向等。你当前实验中没有卖出信号的止损逻辑（因为卖出即平多），且参数 `stop_loss_c` 需要调节。
